데이터 적재 — 5종 변환기 → 단일 schema

Word · Excel · PPT · MD · PDF → 표준 JSON → PostgreSQL 적재. 자동 임베딩 인덱싱(e5_base 768차원).

적재 흐름

STEP 1
원본 파일 (.docx / .xlsx / .pptx / .md / .pdf)
VSCode 확장 Upload 탭에 드래그&드롭, 또는 사전 변환된 ZIP bundle.
STEP 2
서버 측 5종 변환기
포맷별 converter (heading 인식 / 표 추출 / 첨부 분리). 결과: 표준 JSON.
STEP 3
PostgreSQL 적재
records (메타) + record_sections (계층 청크) + record_attachments (첨부).
STEP 4
자동 임베딩
AUTO_EMBED_ON_INSERT=true → e5_base 가 각 섹션을 768차원 벡터로. 시맨틱 검색 즉시 가능.

startup 검증

EMBEDDING_DIM consistency check

main.py lifespan 단계에서 pg_attribute 조회로 실제 vector(N) 컬럼 차원과 EMBEDDING_DIM 환경변수 일치 여부 검증. 불일치 시 부팅은 진행하지만 WARNING 로그 → 임베딩 쓰기 실패 사전 차단.

모든 임베딩 컬럼은 자동 매핑 (e5_small=384 / e5_base=768 / e5_large=1024) — start_api.sh 가 EMBEDDING_PROVIDER 만으로 dim 결정.

SLIDE 08

Agent 시스템 — RAG 레시피 (v0.13.0)

agent = 단순 라우팅 태그가 아니라 "검색·응답 레시피". LLM 선택만 하고 그 뒤 RAG 동작은 서버가 통제.

4가지 신규 컬럼 (Migration 0014)

retrieval_config (JSONB)

top_k · score_threshold · data_type_filter · tag_boost. 검색 동작을 데이터로 통제.

system_prompt (Text)

LLM 에 그대로 주입. {base_url} / {agent_type} / {agent_name} placeholder. 도구 가이드 자동 append (opt-out: <!-- no-tool-guide -->).

response_config (JSONB)

max_tokens · citation_required · refusal_message · refuse_below_score.

sample_queries (Text[])

"이 질문이 오면 이 agent 로" 라우팅 힌트. 자동으로 agent_sample_embeddings 에 임베딩 → recommend_agents 에서 가중 합산.

recommend_agents 점수 (정규화 후)

# Migration 0016 후 동작 agent_record_score[at] += rid_score_sum / candidate_sections # → 정상 범위 [0, 1] agent_sample_score[at] += sample_sim # [0,1] # per-agent cap = 3 (한 agent 가 sample 로 독점 못함) agent_score[at] = agent_record_score[at] + AGENT_SAMPLE_WEIGHT * agent_sample_score[at] # 기본 가중치 = 5.0

실측 효과

시나리오이전 (sum)이후 (정규화)
50 sections, 1 sample 매칭52.0 (sample 8.6%)5.45 (sample 82%)
sample-only agent 라우팅밀림정상 선택

운영자가 sample_queries 만 잘 적어두면 — 코드 수정 없이 — 라우팅을 직접 통제할 수 있다.

RAG 동작 데이터화 = 모델 사이즈 내려도 (Haiku/Qwen) 답변 품질 일관성 유지

Agent 운영 — history · preview · resync · prune

RAG 레시피의 변경 이력·저장 전 검증·라우팅 인덱스 재계산·이력 청소까지 모두 API.

📜 history

append-only audit log. create/update/delete 자동 snapshot. 삭제된 agent 도 이력 조회 가능. UI 에 L/R 라디오 diff 뷰.

GET /api/agents/{type}/history

🧪 preview

저장 전 RAG dry-run. 현재 폼 값(미저장)으로 semantic_search + threshold + 옵션 LLM 호출. OPENAI_API_KEY 없으면 chunks 만 표시 (graceful).

POST /api/agents/preview

🔄 resync

sample_queries 임베딩 강제 재계산. UI 에 stale 칩 (indexed ≠ length) 자동 표시. sync 실패는 try/except + 1회 자동 retry.

POST /api/agents/{type}/resync-samples

POST /api/agents/resync-samples-all (batch)

🧹 prune

per-agent keep_last + 절대 age 기준 청소. agents_history 무한 누적 방지.

POST /api/agents/history/prune?keep_last=50&older_than_days=90

자동화 이미 들어간 곳

changed_by 자동 캡처

VSCode 확장 → X-User-Id: vscode:<username> 헤더 자동 부착 → agents_history.changed_by 채움. 인증 미연동 단계의 임시 식별자.

diff-aware sync

update_agent 가 sample_queries 변경 여부 체크 → 같으면 임베딩 재계산 skip (성능).
