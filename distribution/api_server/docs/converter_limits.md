# 변환기별 알려진 한계

5 개 변환기(Word / Excel / PPT / Markdown / PDF)가 지원하지 않는 기능과 권장 대응을 한눈에 정리.
"왜 내 자료가 깨지는가" 의 1차 답이 여기 있다.

---

## 한 표 요약

| 변환기 | 미지원 / 손실되는 기능 | 권장 대응 |
|---|---|---|
| **Word** (`.docx`) | track changes (수락 전), 텍스트박스 위치, 그림 자르기 정보, 워드아트, 본문 위 floating 그림, 매크로, 도형 그룹 | 표준 양식 작성 — Heading 스타일·캡션·인라인 그림. 변경 추적은 적용/거부 후 저장. |
| **Excel** (`.xlsx`) | 셀 수식(값만), 차트 데이터·서식 의미, 조건부 서식, 피벗테이블 정의, 외부 링크, VBA | `_GLOSSARY` / `_META` 시트로 의미 보완. 차트는 첨부로 별도 등록. |
| **PPT** (`.pptx`) | 애니메이션·빌드 효과, 슬라이드 전환, 발표자 노트(부분), SmartArt 일부, 임베디드 동영상 | 정적으로도 정보가 완전하도록 작성. 차트는 데이터 시리즈가 있으면 표로 추출됨. |
| **MD** (`.md`, `.markdown`) | HTML 임베드 (iframe / script), 사용자 정의 확장 (Pandoc filters 등), Mermaid 다이어그램, footnote 일부 | CommonMark + GFM 만 사용. 다이어그램은 PNG 로 export 후 이미지 참조. |
| **PDF** (`.pdf`) | 스캔 PDF (텍스트 레이어 없음), 다단 레이아웃 정확도, 차트·다이어그램 데이터, 폼 필드, 디지털 서명 검증, 암호화 PDF | **Word→PDF 권장** (직출). 스캔 PDF 는 `--ocr` 옵션 + tesseract 별도 설치. |

---

## 변환기별 상세

### Word (`.docx`)

가장 잘 지원된다. 표준 양식(Heading 스타일 + 인라인 그림 + Word 표 + 캡션)만 지키면 거의 무손실.

**잘 추출되는 것**:
- 제목 1/2/3 → sections[].level
- 본문 단락 → blocks[].paragraph
- Word 표 → tables[] (셀 병합·헤더 인식)
- 인라인 그림 → figures[] + 첨부
- 캡션 ("그림 1 — XX", "표 1 — YY")

**놓치는 것**:
- **Track changes 미적용** — 수락/거부 전이면 변경 텍스트가 본문에 섞임.
- **텍스트박스의 위치** — 텍스트박스 내용은 추출되지만 페이지 레이아웃은 사라짐.
- **그림 자르기 정보** — Word UI 의 "자르기" 표시는 무시되고 원본 그림이 들어감.
- **워드아트** — 텍스트로 추출되지 않음.
- **Floating 그림 (본문과 분리)** — Inline 으로 변경 권장.
- **매크로** — `.docm` 은 매크로 부분 무시.

**대응**: 작성 시 표준 양식 준수. 변경 추적은 검토 완료 후 적용/거부.

---

### Excel (`.xlsx`)

값은 추출되지만 의미는 사라진다. `_META` / `_GLOSSARY` 시트로 보완하는 패턴이 권장.

**잘 추출되는 것**:
- 시트별 표 → tables[] (한 시트에 여러 표가 있어도 빈 행/열 기준으로 분리)
- 셀 값 (날짜·숫자 타입 보존)
- 헤더 행 (자동 감지 또는 첫 행)

**놓치는 것**:
- **수식**: 결과 값만 추출. 수식 자체(`=A1+B1`)는 보존되지 않음.
- **차트**: 차트 객체와 데이터 시리즈는 분리 추출 시도하지만 차트 의미(축 단위·범례·색상)는 사라짐.
- **조건부 서식 / 셀 색상**: 의미 정보로 쓰면 손실.
- **피벗테이블**: 정의가 아니라 결과 셀만 보임.
- **외부 링크 / 데이터 연결**: 끊김.
- **VBA 매크로**: `.xlsm` 은 매크로 부분 무시.

**대응**: `_META` 시트(시트별 의미)·`_GLOSSARY` 시트(컬럼별 의미·단위·범위) 사용.
자세한 표준은 본 프로젝트 루트의 `excel_to_json_conversion_rules.md` 참고.

---

### PPT (`.pptx`)

이번 웨이브에서 차트 추출이 보강됐다. 단, 동적 효과는 모두 손실.

**잘 추출되는 것**:
- 슬라이드별 텍스트 (제목/본문 자리표시자)
- 표 (Word 와 동일 처리)
- 그림 → figures[]
- **차트 데이터 시리즈** (이번 웨이브) — 막대·선·파이 차트의 데이터 표 추출
- 발표자 노트 (대부분)

**놓치는 것**:
- **애니메이션 / 빌드 효과 / 슬라이드 전환**: 정적 슬라이드로 평탄화.
- **SmartArt 의 도식 의미**: 텍스트는 추출되지만 관계 그래프는 사라짐.
- **임베디드 동영상 / 사운드**: 미디어 첨부로만 보존, 자동재생 정보 없음.
- **3D 모델 / 수식 입력**: 텍스트로 변환 안 됨.
- **마스터 슬라이드 디자인**: 본문 추출에는 영향 없으나 디자인 의미 (색상 범례 등) 손실.

**대응**: 정적으로 봐도 슬라이드만으로 정보가 완결되도록 작성. 핵심 다이어그램은 캡션 단 그림으로.

---

### Markdown (`.md`, `.markdown`)

CommonMark + GFM 표준만 사용하면 거의 무손실. 가장 깨끗한 입력 포맷.

**잘 추출되는 것**:
- ATX/Setext 헤더 → sections[]
- GFM 표 → tables[]
- 코드 블록 (fenced + 언어 지정) → blocks[].code
- 이미지 (![]() ) → figures[]
- 링크·강조·리스트

**놓치는 것**:
- **HTML 임베드**: `<iframe>`, `<script>`, `<style>`, `<svg>` 등은 무시 또는 문자열로만 보존.
- **사용자 정의 확장**: Pandoc filters, MyST, Obsidian 위키링크(`[[...]]`) 등.
- **Mermaid / PlantUML 다이어그램**: 코드 블록으로만 보존, 렌더링 결과는 없음.
- **Front matter** (`---` YAML 블록): 일부 파서만 인식. 안전하게 본문 메타로 사용 금지.
- **각주 (footnote)**: GFM 한정 일부 지원.

**대응**: 다이어그램은 PNG 로 export 후 이미지로 참조. front matter 대신 적재 시 메타 폼 사용.

---

### PDF (`.pdf`)

가장 정보 손실이 큰 포맷. 가능하면 Word→PDF 직출(텍스트 레이어 보존)을 사용한다.

**잘 추출되는 것** (텍스트 레이어가 있을 때):
- 본문 텍스트 (단일 컬럼 기준 안정)
- 표 (간단한 격자 표)
- 그림 (raster 추출)

**놓치는 것**:
- **스캔 PDF (이미지)**: 텍스트 레이어 없음 → OCR 폴백 필요 (`--ocr` 옵션 + tesseract 별도 설치).
  자세한 절차는 [`FAQ.md` Q5](FAQ.md) 참고.
- **다단 레이아웃**: 2 단·3 단 컬럼은 읽는 순서가 뒤섞일 수 있음.
- **차트 / 다이어그램 데이터**: 그림으로만 보존, 데이터 추출 불가.
- **폼 필드**: 채워진 값은 별도 추출. 필드 정의 의미는 사라짐.
- **디지털 서명**: 검증되지 않음 (서명 메타만 첨부 메타에 표시).
- **암호화 PDF**: 비밀번호 보호된 파일은 변환 실패. 사전에 해제.

**대응**:
- 가능하면 **Word → PDF 직출** (Acrobat 또는 Word "PDF/XPS 만들기"). 스캔 금지.
- 다단 자료는 단일 단으로 재출력.
- OCR 결과는 일반 추출보다 **품질이 낮다** — 가능하면 원본 .docx 를 적재하고 PDF 는 첨부만.

---

## OCR 선택 기준 빠른 표

| 자료 종류 | 권장 적재 방법 |
|---|---|
| 본인이 Word 로 쓴 자료 | **`.docx` 그대로 적재** (PDF 변환 불필요) |
| 외부에서 받은 PDF (텍스트 레이어 있음) | PDF 그대로 적재 OK |
| 스캔본 PDF (옛 자료 / 손글씨 보고서) | `--ocr` 옵션 + tesseract 한글 언어팩. **OCR 품질 검증 후 적재** |
| 사진으로 찍은 종이 보고서 | 사진 → OCR → Word 정리 → Word 적재 (권장 흐름) |
| 그림이 핵심인 자료 (회로도 / CAD 도면) | 본문 자료를 .docx 로 만들고, 도면은 첨부로 |
