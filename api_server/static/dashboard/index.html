시작 가이드 — 표준 절차

데이터 적재부터 AI가 그 데이터를 쓰기까지의 전체 흐름. 처음이면 위에서 아래로 따라가세요.

↗ 전체 절차서
1

데이터 적재

VSCode Extension의 Upload 탭에 파일을 드래그(.docx .pdf .pptx .xlsx .md) 또는 Bundle 탭에 zip.

  • 필수 메타: team(대문자 2~4자, org에 등록) · group(대문자 2~5자) · year
  • seq 비우면 자동 증가 — 기존 레코드 안 덮어씀
  • 폼의 tags/agents는 본문 추출 태그와 합쳐서 보존됨
  • 계층 데이터면 parent_record_id(campaign id) 입력 — 비워도 업로드 후 추천받아 연결 가능

확인: 이 대시보드 카탈로그 / 분석 탭 또는 /api/discover

2

Agent 정의

Extension Agents 탭 → "New agent". 또는 "LLM 초안 생성"(데이터군 태그/타입 한정 → 폼 자동 채움 → 검토·수정).

  • 핵심: common_tags · data_types · retrieval_config(top_k/score_threshold/max_depth) · system_prompt · sample_queries
  • 기존 agent 수정은 목록의 Edit (agent_type만 고정) — 이력은 자동 보존
  • 저장 전 "Test preview"로 실제 검색·답변 미리보기
3

데이터 ↔ Agent 연결

업로드 시 agents= 직접 지정, 또는 Agent 저장 시 "저장 후 매칭 레코드 자동 바인딩" 체크.

  • 매칭: required_doc_type + data_types + (required_tags 전부 / 없으면 common_tags 겹침)
  • 계층: 업로드 결과 화면의 "유사 부모(campaign) 연결" → 후보 표 확인 → 한 클릭 연결
4

Agent 활성화 가장 중요

DB만 쌓이면 LLM은 모릅니다. Extension Console 탭에서:

  • agent 선택 → AI 클라이언트 선택(Cline/Claude Desktop/Claude Code/Cursor/Copilot/Gemini/Codex)
  • "1,2,3 불러오기" → "자동 설치 (MCP + Prompt)"
  • → MCP 서버 등록 + 그 agent의 system_prompt가 클라이언트 룰 파일에 주입
5

런타임 사용 (LLM 측)

설치된 클라이언트 새 세션에서 자동 흐름:

  1. get_agent_session(agent_type) — persona·설정 초기화
  2. agent_search(agent_type, q) — retrieval_config 자동 적용 검색
  3. get_record_sections(record_id) — 상세 RAG 청크
  4. 답변마다 (source: <record_id> §<section>) 인용
6

운영 루프

  • 새 데이터 → 해당 agent bind-matching 재실행
  • sample_queries 변경 → resync-samples
  • 설정 튜닝 → Agents Edit → 필요 시 클라이언트 재-자동설치
새 서버 셋업 (2단계)
  • 0) sudo bash bootstrap.sh — 시스템 의존성(apptainer/node20/python3.12-venv) 1회. idempotent — MXWhitePaper 선행 시 거의 no-op
  • 1) bash setup.sh → PG+API 기동 · Extension 빌드·게시 · (code CLI 있으면) 자동 설치 · 대시보드 오픈
  • 상태 점검: bash status.sh / 상세: bash diag.sh --tail-logs
  • Extension 받기: 이 대시보드 상태 탭 → VSCode Extension 카드 → Download .vsix
