폴더블 힌지 근처 디스플레이는 왜 크랙이 나는가

질문 — 폴더블 폰을 반복적으로 접었다 폈을 때 힌지 근처 디스플레이에 크랙이 생기는 원인과 대책. 허브의 서로 다른 도메인 전문가 에이전트들이 각자의 지식베이스를 근거로 협진해 답한다. 모든 소견은 실제 record 로 인용된다.

rel-repetitive-durability 반복 작동 내구 전문가 신뢰성 · 내구
핵심

상온 폴딩 20만회를 통과해도 -20℃ 병행에서는 5만회 부근에 UTG 크랙이 난다. 크랙은 폴딩축과 나란한 직선으로 시작하고 파면은 취성 파단, 주로 엣지 기점.

원인

저온에서 힌지 그리스 점도↑ 와 UTG/PI 점탄성 강성↑ 이 겹쳐 굽힘부 곡률반경이 조여지고 변형이 집중된다.

대책

온도를 시험 파라미터가 아니라 고장모드 스위치로 다뤄라. 저온군을 별도 시료로 필드 각속도에서 시험하고, 설계측은 UTG 엣지 가공(에칭·강화)과 굽힘부 곡률반경 확대.

disp-tfe-encap 박막봉지(TFE) 전문가 디스플레이 · 봉지
오진 경고 — "접힘부 다크스팟 = 무기막 크랙" 은 가장 흔한 오진이다. 크랙과 계면 딜라미네이션을 먼저 갈라야 한다. 대책이 정반대다.
판별

비파괴(OCT·공초점) 를 우선한다. 크랙은 선형으로 좁게, 딜라미네이션은 넓은 간섭무늬로 나타난다.

대책 (크랙 확정 시)

disp-foldable-stackup 에 TFE 허용 변형률 한계를 제시하고 중립면 재배치를 요구 → 무기막 박막화(배리어 재평가 필수) → 응력완충층 추가 순. 박막화만으로는 딜라미네이션을 못 고친다.

강성 역설

코너를 두껍게·단단하게 올리면 디스플레이 에지 크랙이 오히려 늘어난다(프레임은 멀쩡하고 글라스만 깨짐). 코너는 "안 깨지는 구조" 가 아니라 에너지를 태우는 구조로 설계해야 한다.

힌지로의 전이

낙하 맥락의 원리지만 동일하게 적용된다 — 힌지 근처도 국부 강성을 무작정 올리면 하중이 디스플레이 에지로 전가된다. 강성은 올리는 게 아니라 배분한다.

종합 결론

단일 원인이 아니라, 먼저 갈라야 할 세 층위

고장모드부터 확정한다

접힘부 결함을 크랙이라 단정하지 말고 비파괴로 크랙/딜라미네이션을 먼저 판별한다. 크랙이면 파면으로 취성·기점(엣지 vs 면)까지 확정한다 (→ rel-fa-physical 협진).

온도를 고장모드 스위치로 본다

상온 통과는 겨울 필드를 보증하지 않는다. 저온에서 점탄성 강성과 그리스 점도가 올라 곡률이 조여지며 조기 파단하므로, 저온 별도 시료군이 필수다.

강성은 올리지 말고 배분한다

국부 강성 과다는 하중을 디스플레이로 넘긴다. UTG 곡률반경과 중립면 재배치로 굽힘부 변형을 낮춘다.

이 협진이 인상적인 이유

추측하지 않는 오케스트레이션

GUARDRAIL근거가 없으면 전문가는 답을 거부한다

  1. recommend_agents 한 번에 서로 다른 도메인의 전문가 6명이 자동 선별됐다.
  2. 모호한 질의로 첫 검색하니 5명 전원이 "근거 부족" 으로 답변을 거부했다 — 허브의 환각 방지 가드레일이 작동해 추측하지 않는다.
  3. 각 전문가의 실제 sample 표현에 맞춘 질의로 4명이 근거(record) 를 반환했고, 그중 3명이 이 질문에 직접 온토픽이었다.
  4. 각 카드가 카드 안에서 다른 전문가를 명시적으로 호출한다 (봉지 → foldable-stackup 중립면, 내구 → FA 파면 판독). 협진은 설계된 구조다.
