01 개요 — 무엇을 푸는가

사업부 곳곳에 흩어진 엔지니어링 데이터를 한 곳에 모으고, 자연어로 물으면 실제 레코드를 근거로 인용해 답하게 만든 서버다. 두 가지 질문에 대한 답이 설계의 축이다.

Q1 · 수집

어떻게 DB를 모으나

드래그&드롭 표/문서를 Claude가 파싱해 넣고(import_record), 오피스 파일은 정밀 변환하며 (convert_file), 대시보드 업로드·외부 소스 동기화까지 4경로. 넣기 전 유사 데이터를 찾아 일관 분류를 유도한다.

Q2 · 검색

어떻게 찾나

정확 태그·전문검색·의미(임베딩)·하이브리드(RRF) 4모드에, 에이전트 페르소나의 검색 설정을 자동 적용하는 agent_search. 답변은 항상 record id로 인용한다.

설계 원칙

독립 실행 · 상태 없음

Apptainer rootless로 어떤 서버에서도 격리 실행. MCP 서버는 stateless — 행동의 주체는 클라이언트 LLM이고, 서버는 근거(데이터)와 규약만 제공한다.

핵심 개념 — 이 허브는 "행동하는 에이전트"가 아니라 검색용 페르소나 + 근거 데이터의 카탈로그다. 개인 PC의 Claude가 MCP로 붙어, 데이터를 넣을 때는 일관 분류를 지키도록 유도받고, 찾을 때는 페르소나 규약에 따라 출처를 인용해 답한다.

런타임

Apptainer rootless

PG는 Apptainer 인스턴스, API는 native venv. root 없이 어떤 서버에서도 격리 기동. boot.sh가 재부팅 복구, watchdog가 매분 죽은 컴포넌트만 되살린다.

임베딩

e5-base 768d

E5 비대칭 인코딩 — 질의는 query:, 문서는 passage: 프리픽스. 섹션 임베딩(검색용)과 시그니처 임베딩(유사 분류용)을 분리 저장.

통합

포털 · 확장 · 동기화

HWAX 포털 서브패스(/ai-data-hub/), VSCode 확장, SignalForge·MXWP 외부 소스 동기화까지 한 서버에서 물린다.

04 데이터 수집 — 어떻게 모으나

데이터가 들어오는 경로는 네 갈래다. MCP는 바이너리를 못 받으므로, 파일은 Claude가 파싱하거나 서버가 변환하거나 대시보드가 업로드한다. 공통으로, 넣기 전에 유사 데이터를 찾아 일관 분류를 유도한다.

경로 A

import_record

드래그&드롭한 표/문서를 Claude가 구조화(headers/rows/섹션)해 저장. dry-run 지원.

경로 B

convert_file

docx·xlsx·pdf·pptx를 서버가 정밀 변환 → record. 인박스 경로, 경로탈출 차단.

경로 C

대시보드 업로드

/api/mcp_tools/upload·/api/convert — 사람이 직접 파일 업로드.

경로 D

외부 소스 동기화

/api/sync — SignalForge·MXWP 등 외부에서 주기적으로 끌어옴.

일관 분류 규약 — 대량으로 쌓여도 서로 찾히게

임의의 데이터가 들어올 때, 추측으로 분류하면 나중에 못 찾는다. 그래서 import_record 전에 반드시 다음을 거친다.

  1. find_similar_data — headers/caption으로 기존 유사 레코드를 임베딩 유사도로 찾는다.
  2. AI가 채울 제안 — doc_type·tags·graph_type은 유사 레코드 기준으로 제안(다수결/합집합). 검토 후 채운다.
  3. 사람이 정할 것 — team·group은 자동 확정 금지. 후보를 보여주고 "어느 팀/그룹인가요?" 물어 사람이 정한다. (유사도가 높아도 다른 팀 데이터일 수 있어, 오분류는 데이터를 영영 못 찾게 만든다.)
  4. 유사 데이터가 없으면(confidence=low/none) 추측하지 말고 describe_record_schema로 가능한 값을 보여주며 직접 묻는다.
왜 규칙표가 아니라 임베딩 유사도인가 — 선언적 규칙표는 데이터가 수만 건으로 늘면 유지가 불가능하다. "비슷한 건 비슷하게 분류"를 임베딩 유사도로 자동 유도하면, 자동/대량 주입에서도 분류가 스스로 수렴한다. 팀/그룹만은 예외로 사람이 결정한다.

멱등성 · content_hash 중복 차단 · external_id_map UPSERT · deleted_at 소프트삭제 · 시그니처 임베딩 매분 백필

05 검색 — 어떻게 찾나

네 가지 검색 모드가 있고, 그 위에 에이전트 페르소나가 검색 설정을 자동 적용한다. 정상 흐름은 "에이전트 추천 → 세션 초기화 → 에이전트 검색 → 섹션 인용"이다.

페르소나 채택은 클라이언트 책임 — MCP는 stateless라 서버가 세션 규칙을 자동 교체하지 못한다. get_agent_session이 준 system_prompt를 그 시점부터 지배 규칙으로 삼고, 모든 답변을 source: DATA-HE-CAE-2026-…​ §4 형태로 인용한다.

표(DATA) 위에서 바로 집계

data_aggregate로 저장된 표에서 avg·max·min·sum·count를 group_by/where와 함께 즉시 계산한다(REST /api/data/{id}/aggregate와 동일 로직). 컬럼·행 조회는 /api/data/{id}/columns·/rows.

06 MCP 도구 26종

Claude Desktop 연결

MCP 서버는 http://<HOST_IP>:8001/mcp/에 마운트(stateless_http). Claude Desktop MCP 설정에 HTTP 커넥터로 등록하면 26개 도구가 노출된다. 외부 호스트 접속 시 FastMCP의 DNS rebinding 보호 때문에 Host 화이트리스트/해제가 필요할 수 있다.
