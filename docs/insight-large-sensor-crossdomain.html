교차심문이 만든 인사이트

증상 하나가 원인 하나에 대응하지 않는다 — 증상↔원인은 다:다다. 그래서 한 대책으로는 절반만 잡힌다.

"워블" = 두 독립 원인
OIS 서보 포화(질량) + 롤링셔터 스큐(라인타임). OIS만 강화하면 워블이 절반만 준다.
"프리뷰 끊김" = 두 축
시간축(라인타임 예산) + 메모리축(DRAM 대역폭). ISP 클럭만 올려선 안 잡힘.
"낙하 AF" = degrades + reveals
질량×관성(새 결함) + 조립 산포(기존 결함 드러냄). reveals가 "일부 세트만"의 정체. 대책을 두 축으로 분리.

진단 한 수 · 단일 촬영 매트릭스로 4도메인 분리

크롭·비닝(유효라인 절반) × OIS on/off × 노출 2단 + 온도·DRAM 큐 로깅
  • 크롭으로 워블·끊김·노이즈 동반 감소 → 리드아웃 라인타임/처리량 축
  • 크롭 후 남는 워블이 OIS on/off로 갈림 → 질량 기반 OIS 축
  • 노출 줄여도 스큐 불변 → 롤링셔터 확정
  • DRAM 큐 포화 + ISP 클럭 스윕 무반응 → 대역폭 병목 확정
  • 낙하 AF는 질량항이라 이 매트릭스서 불변 → 자동 분리. 낙하 전후 스루포커스 Δ로 reveals/degrades 마저 나눔

충돌 · 서로 당긴다

  • OIS 게인↑ ↔ 워터베드·공진 — 게인 밀면 수백 Hz 증폭, 고질량은 공진↓로 노치 여유 감소
  • 크롭·비닝 ↔ 대형센서 목적 — 스큐·발열 줄이면 유효 화소를 스스로 깎음
  • AF 감도(k↓) ↔ 낙하 내성 — 자세차 Δx=m·g/k와 낙하 취약성 동반 증가
  • 유연 서스펜션 ↔ 강성 스토퍼 — OIS 스트로크 vs 낙하 지지, 접착 설계로만 절충
정직성 · 엣지도 근거 없으면 가설로

워크플로가 그럴듯하지만 근거 없는 커플링 4개를 사실이 아니라 가설로 분리했다 — 교차도메인은 환각 위험이 가장 크다.

  • 열-OIS 커플링(발열이 억제율을 낮춰 워블↑) — 근거 없음
  • 대역폭-제어 지연(DRAM 포화가 자이로 처리 지연) — 근거 없음
  • 발열-접착(4K 발열이 낙하 전 틸트 마진 잠식) — 직접 연결 근거 없음
  • 열-초점 드리프트(센서 열팽창이 초점면 이동) — 근거 없음
